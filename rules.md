# Project Context
We are building a highly aesthetic, static, dependency-free academic blog post to be hosted on GitHub Pages. The content is a Mechanistic Interpretability AI research writeup.

The user will provide two things:
1. `writeup.md` (the text content, which includes raw HTML tags, LaTeX math, and references to local diagrams).
2. A `diagrams/` folder containing interactive HTML visualizations and SVGs.

Your job is to build a minimal build system and CSS architecture that converts the markdown into a production-ready, beautiful static site.

# Design System & Aesthetics
The aesthetic is "constraint as aesthetic." It should feel like a high-quality printed paper or vintage journal. No flashy animations, no shadows, no rounded corners. 
- **Typography:** "EB Garamond" (or generic serif) for prose. "JetBrains Mono" (or monospace) for code, metadata, and math.
- **Palette:** 
  - Background: `#F7F5F0` (warm cream)
  - Main Text: `#2A2A2A` (soft charcoal)
  - Headers/Emphasis: `#111111` (heavy ink)
  - Borders/Muted Text: `#7A766C`
  - Code Background: `#EBE7DD`
- **Layout:** Max-width of 720px for optimal reading measure. Centered. Generous line height (1.6 to 1.8). Paragraphs should have a slight bottom margin.
- **Visuals:** Any `<img>` or `<iframe>` must have a 1px solid `#7A766C` border. 

# Directory Structure to Create
```text
/
├── package.json         # (Minimal, just for the markdown parser dependency)
├── build.js             # (The custom static generator script)
├── writeup.md           # (USER PROVIDED - Do not overwrite)
├── diagrams/            # (USER PROVIDED - Do not overwrite, containing .html and .svg files)
├── src/
│   ├── template.html    # (The HTML skeleton with CSS/Font links)
│   └── style.css        # (The design system implementation)
└── docs/                # (The output directory for GitHub Pages)
Implementation Steps
Step 1: Initialize Project
Initialize a basic package.json.
Install marked (for markdown parsing).
Ensure docs/ is set up as the output folder (GitHub Pages uses /docs out of the box).
Step 2: Write src/style.css
Implement the exact design system described above.
Ensure blockquote has a left border matching the charcoal color, with italicized text.
Ensure tables (for the data comparisons) have clean, 1px bottom borders on rows, left-aligned text, and monospace fonts for numerical data.
Ensure iframes (for the HTML diagrams) are responsive (width: 100%, scalable height).
Step 3: Write src/template.html
Include the CDN links for Google Fonts (EB Garamond, JetBrains Mono).
Include the CDN script for MathJax or KaTeX to render LaTeX equations found in the markdown.
Include a <main> container where the parsed HTML will be injected.
Include the hardcoded title: "SaeLit: Finding Guilt in the Weights".
Step 4: Write build.js
Write a Node.js script that:
Reads writeup.md.
Configures marked to allow raw HTML (crucial, because the user will use <iframe> tags to embed their HTML diagrams).
Parses the Markdown into HTML.
Reads src/template.html and replaces a placeholder (e.g., {{CONTENT}}) with the parsed HTML.
Copies src/style.css and the entire diagrams/ directory into the /docs folder.
Writes the final HTML to docs/index.html.
Rules for the AI
Do NOT use React, Next.js, Tailwind, or any heavy frontend frameworks.
Do NOT alter or generate writeup.md. Assume it exists.
Ensure the CSS correctly targets tables and code blocks generated by the markdown parser.
Log a success message when node build.js runs successfully, instructing the user to commit and push to GitHub.
